# Yggdrasil

简单介绍一下 Model文件夹下各个类的作用

User类和Book类是很典型的模型类，用来对具体的用户/书建模，它们拥有的成员变量就是它们在数据库中的个字段


DatabaseUtility类是专门用来进行与数据库交互的类。
其中有大量的静态方法用来查询/修改/增加数据
一般而言，这些静态方法的返回值是int类型，用来表示状态码，1代表正常结束，-1代表出错，这个错误一般就是数据库连接失败，有可能是因为没有网络连接。


Global类用来存放某些经常使用的变量，等于是一个存放全局变量的地方，目前只有一个变量user，用来存放当前登录的用户信息。这个变量应该在登录界面登陆成功之后立即更新


Testing类是用来测试DatabaseUtility类中的方法的，其实就是把相关信息在控制台打印出来，检查一下。
